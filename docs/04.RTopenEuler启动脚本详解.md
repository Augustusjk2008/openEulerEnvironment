# RTopenEuler启动脚本详解

## 概述

RTopenEuler系统的启动脚本负责从内核初始化到用户服务的完整启动流程。本文档基于典型的RTopenEuler系统启动脚本(rc5.d和rcS.d目录)，详细分析RTopenEuler启动机制的各个阶段和实现细节。

## 目录结构

### rcS.d - 系统初始化阶段(S级)
- **功能**：系统基础环境初始化
- **脚本数量**：15个（按数字前缀排序执行）
- **执行时机**：系统启动的最早期阶段

### rc5.d - 运行级别5(多用户图形界面模式)
- **功能**：网络服务和系统功能启动
- **脚本数量**：18个（按数字前缀排序执行）
- **执行时机**：系统初始化完成后

## 第一阶段：系统初始化(rcS.d)

### S02banner.sh - 系统启动横幅

```bash
#!/bin/sh
### BEGIN INIT INFO
# Provides:          banner
# Required-Start:    $local_fs $remote_fs $syslog $network
# Required-Stop:
# Default-Start:     S
# Default-Stop:
# Short-Description: Display boot banner
### END INIT INFO

echo "Please wait: booting..." > /dev/console

# 确保控制台设备存在
if [ ! -c /dev/tty ] ; then
    mknod -m 666 /dev/tty c 5 0
fi
```

**功能解析**：
- 在控制台输出启动信息，告知用户系统正在启动
- 确保/dev/tty设备存在，这是基本的控制台设备
- 为用户提供视觉反馈，增强用户体验

### S02sysfs.sh - 挂载内核虚拟文件系统

```bash
#!/bin/sh
### BEGIN INIT INFO
# Provides:          sysfs
# Required-Start:    $local_fs $remote_fs $syslog $network
# Required-Stop:
# Default-Start:     S
# Default-Stop:
# Short-Description: Mount kernel virtual filesystems
### END INIT INFO

# 挂载proc文件系统
mount -t proc proc /proc

# 挂载sysfs文件系统
mount -t sysfs sysfs /sys

# 挂载debugfs文件系统
mount -t debugfs debugfs /sys/kernel/debug
```

**功能解析**：
- **proc**：提供进程和系统信息的虚拟文件系统
- **sysfs**：内核设备的属性和属性值
- **debugfs**：内核调试信息的接口
- **configfs**：运行时系统配置接口

### S04mdev - 设备管理器

```bash
#!/bin/sh
### BEGIN INIT INFO
# Provides:          mdev
# Required-Start:    $local_fs $remote_fs $syslog $network
# Required-Stop:
# Default-Start:     S
# Default-Stop:
# Short-Description: Manage dynamic devices
### END INIT INFO

# 启动mdev守护进程
echo /sbin/mdev > /proc/sys/kernel/hotplug

# 扫描并创建所有设备
mdev -s

# 执行冷插拔模块加载
for i in /sys/class/mdev/devices/*; do
    mdev -s $i 2>/dev/null
done
```

**功能解析**：
- **mdev**：busybox中的轻量级设备管理器，替代udev
- **hotplug**：设置内核热插拔事件的处理程序
- **-s**：扫描所有已存在的设备并创建设备节点
- **冷插拔**：处理系统启动时已存在的设备

### S05checkroot.sh - 根文件系统检查

```bash
#!/bin/sh
### BEGIN INIT INFO
# Provides:          checkroot
# Required-Start:    $local_fs $remote_fs $syslog $network mdev
# Required-Stop:
# Default-Start:     S
# Default-Stop:
# Short-Description: Check and repair root filesystem
### END INIT INFO

# 检查是否设置了fastboot选项
if [ -f /fastboot ]; then
    echo "Fastboot: skipping root filesystem check"
    exit 0
fi

# 检查是否设置了forcefsck选项
if [ -f /forcefsck ]; then
    echo "Forcing root filesystem check"
    fsck_options="-f"
else
    fsck_options="-a"
fi

# 执行文件系统检查
fsck $fsck_options /
```

**功能解析**：
- **fastboot**：跳过文件系统检查，加快启动速度
- **forcefsck**：强制执行文件系统检查
- **fsck**：文件系统检查和修复工具
- **依赖mdev**：确保设备节点已创建后再执行检查

### S06modutils.sh - 模块工具（关键脚本）

```bash
#!/bin/sh
### BEGIN INIT INFO
# Provides:          modutils
# Required-Start:    $local_fs $remote_fs $syslog $network
# Required-Stop:
# Default-Start:     S
# Default-Stop:
# Short-Description: Load kernel modules
### END INIT INFO

# 加载/etc/modules中的模块
if [ -f /etc/modules ]; then
    while read module; do
        case $module in
            \#*) ;;
            "") ;;
            *) modprobe $module ;;
        esac
    done < /etc/modules
fi

# 加载/etc/modules-load.d中的模块
for file in /etc/modules-load.d/*.conf; do
    if [ -f "$file" ]; then
        while read module; do
            case $module in
                \#*) ;;
                "") ;;
                *) modprobe $module ;;
            esac
        done < "$file"
    fi
done

# 特殊处理：XDMA模块重载
if lsmod | grep -q "xdma"; then
    echo "Reloading XDMA module..."
    rmmod xdma
    modprobe xdma
fi

# 应用启动逻辑
# [此处可能包含特殊应用的启动代码]
```

**功能解析**：
- **模块加载**：从配置文件加载内核模块
- **模块管理**：支持多种配置格式的模块加载
- **XDMA处理**：特殊硬件加速模块的重载机制
- **应用集成**：在模块加载后启动特定应用

### S29read-only-rootfs-hook.sh - 只读根文件系统处理

```bash
#!/bin/sh
### BEGIN INIT INFO
# Provides:          read-only-rootfs
# Required-Start:    $local_fs $remote_fs $syslog $network
# Required-Stop:
# Default-Start:     S
# Default-Stop:
# Short-Description: Handle read-only root filesystem
### END INIT INFO

# 检查根文件系统是否为只读
if ! mount | grep -q " / .*[(,]ro[,)"; then
    exit 0
fi

# 为/var/lib创建overlayfs
mkdir -p /var/lib/overlayfs
mkdir -p /mnt/overlayfs-upper
mkdir -p /mnt/overlayfs-work

mount -t overlay overlay -o lowerdir=/,upperdir=/mnt/overlayfs-upper,workdir=/mnt/overlayfs-work /var/lib/overlayfs
```

**功能解析**：
- **overlayfs**：联合文件系统，实现只读根文件系统的读写层
- **分层存储**：lowerdir（只读层）+ upperdir（读写层）
- **工作目录**：用于管理overlayfs的元数据
- **灵活存储**：在/var/lib下提供可写空间

## 启动脚本的核心机制

### 1. 依赖管理

```bash
### BEGIN INIT INFO
# Provides:          service-name
# Required-Start:    $local_fs $remote_fs $syslog $network
# Required-Stop:
# Default-Start:     S 2 3 4 5
# Default-Stop:      0 1 6
# Short-Description: Brief description
# Description:       Detailed description
### END INIT INFO
```

**依赖声明**：
- **Provides**：声明提供的服务名称
- **Required-Start**：声明依赖的服务
- **Default-Start**：默认启动的运行级别
- **Default-Stop**：默认停止的运行级别

### 2. 执行顺序

```bash
# 按数字前缀排序执行
S02banner.sh
S02sysfs.sh
S03mountall.sh
S04mdev
S05checkroot.sh
...
S99rmnologin.sh
```

**排序规则**：
- 相同数字按字母顺序执行
- 不同数字按数字大小执行
- 使用update-rc.d管理启动顺序

### 3. 错误处理

```bash
#!/bin/sh -e

# 设置严格模式
set -e  # 任何命令失败立即退出
set -u  # 使用未定义变量时报错
set -o pipefail  # 管道中任何命令失败都返回失败
```

**错误处理机制**：
- **严格模式**：确保脚本在错误时立即退出
- **错误检查**：验证关键操作的结果
- **恢复机制**：提供错误恢复的路径

### 4. 配置管理

```bash
# 从配置文件读取配置
if [ -f /etc/default/service ]; then
    . /etc/default/service
fi

# 使用环境变量
CONFIG_VALUE=${CONFIG_VALUE:-default_value}
```

**配置管理**：
- **默认配置**：/etc/default目录下的配置文件
- **环境变量**：支持通过环境变量覆盖配置
- **用户配置**：允许用户自定义配置

### 5. 服务控制接口

```bash
# 支持start-stop-daemon
```

**服务控制**：

## RTopenEuler启动脚本的特色

### 1. 轻量级设计


**轻量级特点**：
- **mdev**：轻量级设备管理器
- **busybox**：集成多种工具的二进制文件
- **资源优化**：最小化内存和CPU占用

### 2. 只读文件系统支持

```bash

# 使用PID文件管理
PID_FILE="/var/run/service.pid"

# 检查服务是否运行
is_running() {
    [ -f "$PID_FILE" ] && kill -0 $(cat "$PID_FILE") 2>/dev/null
}

# 优雅停止
graceful_stop() {
    if is_running; then
        kill -TERM $(cat "$PID_FILE")
        # 等待优雅停止
        for i in $(seq 1 10); do
            if ! is_running; then
                break
            fi
            sleep 1
        done
        # 强制停止
        if is_running; then
            kill -KILL $(cat "$PID_FILE")
        fi
    fi
}
```
