### core/auth_manager.py

**当前状态**: 已测试，覆盖率98.11%

| 优先级 | 问题类型 | 描述 | 建议 |
|--------|----------|------|------|
| 中 | 不可达代码 | 第89行邀请码长度检查无法触发，因为INVITE_CODE固定为16字符 | 如果邀请码长度固定，可以移除此检查；如果支持变长邀请码，需要添加相应测试用例 |

**详细说明**:
```python
# 当前代码 (src/core/auth_manager.py:89)
if len(invite_code) != 16:
    return False, "邀请码长度错误"
```

- 选项1: 移除此冗余检查
- 选项2: 如果未来支持变长邀请码，保留检查并添加变长邀请码的测试用例

### core/config_manager.py

**当前状态**: 已测试，覆盖率93.51%

| 优先级 | 问题类型 | 描述 | 建议 |
|--------|----------|------|------|
| 中 | 环境依赖 | 第28行get_program_dir()中frozen情况需要PyInstaller环境 | 建议在CI/CD流程中添加打包后的测试，或使用依赖注入使该分支可测试 |
| 低 | 异常处理 | 第97-99行IOError处理需要模拟文件系统错误 | 建议使用unittest.mock模拟文件操作以测试此分支 |

**详细说明**:

**问题1: PyInstaller环境依赖 (第28行)**
```python
# 当前代码 (src/core/config_manager.py:28)
if getattr(sys, 'frozen', False):
    return os.path.dirname(sys.executable)
```
此分支仅在PyInstaller打包后的环境中执行，常规测试环境无法覆盖。建议：
- 在CI/CD流程中添加打包后的集成测试
- 或重构为依赖注入模式，使程序目录可通过参数传入

**问题2: 文件系统错误模拟 (第97-99行)**
```python
# 当前代码 (src/core/config_manager.py:97-99)
except IOError as e:
    print(f"保存配置文件失败: {e}")
    raise
```
此异常处理需要模拟文件系统错误（如磁盘满、权限不足等）。建议使用`unittest.mock`模拟文件操作。
